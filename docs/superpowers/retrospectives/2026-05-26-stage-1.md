## Surprises encountered

| # | Surprise | Resolution |
|---|---|---|
| 1 | `terraform 1.9.8` rejects `use_lockfile = true` (TF 1.10+) | switched to `dynamodb_table` matching existing pattern |
| 2 | EKS nodes have taints (`workload-type=platform`, `node-role=system-critical`) | tolerations added to ESO, Atlantis, and ArgoCD helm values |
| 3 | ESO not pre-installed on hub | helm install added as bootstrap step (saved to `k8s/system/external-secrets-bootstrap/`) |
| 4 | ExternalSecrets `v1beta1` deprecated in ESO 2.5.0 | switched to `v1` (CSS + ES manifests) |
| 5 | Atlantis fails to start without `--write-git-creds` flag for GitHub App auth | added flag to deployment.yaml |
| 6 | CF VPC Origin `https-only` + ALB AWS DNS → SNI mismatch | set CF origin `DomainName=atlantis.atomai.click` (matches wildcard cert) — initially didn't work due to missing SG |
| 7 | Source-SG explicit ingress required on ALB SG (CIDR alone insufficient for CF VPC Origin ENI) | added `security_groups = [cf_vpc_origin_sg_id]` ingress rule |
| 8 | `kubectl delete ns argocd` stuck on 56 Application finalizers | stripped finalizers via `kubectl patch ... '{"metadata":{"finalizers":[]}}'` |
| 9 | Helm install fails because old CRDs and ClusterRoles still exist without helm ownership labels | adopted CRDs via labels/annotations; deleted ClusterRoles (helm recreates) |
| 10 | ArgoCD `repo is empty` cache after initial push (was empty when first observed) | `kubectl rollout restart deployment/argocd-repo-server` |
| 11 | Existing CF distribution already using `argocd.atomai.click` alias | imported the existing distribution into Terraform state |
| 12 | Workload sync on spokes shows SharedResourceWarning(105) | Expected — old ArgoCD's resources still in clusters without new owner annotations; ServerSideApply mitigates |
